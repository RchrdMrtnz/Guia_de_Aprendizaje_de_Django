15. Autenticación y Recuperación de Contraseña

Implementar un sistema de autenticación seguro y moderno es vital. En esta sección configuramos el Login y la recuperación de contraseña vía email.

Implementación Técnica

Django provee el módulo django.contrib.auth que maneja gran parte del trabajo pesado. Nosotros nos enfocamos en personalizar la presentación y configurar el envío de correos.

1. Configuración de URLs

En el archivo src/curso_django/urls.py, incluimos las rutas de autenticación por defecto:

path("accounts/", include("django.contrib.auth.urls")),

Esto habilita rutas como /accounts/login/, /accounts/logout/, /accounts/password_reset/, etc.

2. Configuración de Email (Backend)

Para que la recuperación de contraseña funcione, Django necesita saber cómo enviar correos. En un entorno de producción usaríamos un servicio SMTP (como SendGrid o AWS SES), pero para desarrollo, configuramos el Console Backend.

En src/curso_django/settings/local.py:

# Muestra el contenido del correo en la terminal donde corre el servidor
EMAIL_BACKEND = "django.core.mail.backends.console.EmailBackend"

3. Plantillas Modernas con Tailwind

Por defecto, Django busca plantillas en registration/. Hemos creado un set de plantillas personalizadas utilizando Tailwind CSS para lograr un diseño limpio y responsivo.

  • login.html: Formulario de inicio de sesión.
  • password_reset_form.html: Solicitar cambio de contraseña.
  • password_reset_done.html: Confirmación de envío.
  • password_reset_confirm.html: Formulario para nueva contraseña.
  • password_reset_complete.html: Éxito final.

Todas heredan de base_auth.html, asegurando consistencia visual.

Flujo de Usuario (Testing)

Para probar la recuperación de contraseña en tu entorno local:

  1. Ve a /accounts/login/.
  2. Haz clic en "¿Olvidaste tu contraseña?".
  3. Ingresa el correo de tu superusuario (o un usuario creado previamente).
  4. Revisa tu terminal (donde ejecutaste runserver). Verás un bloque de texto simulando el email.
  5. Copia la URL que aparece en la terminal (que empieza con http://127.0.0.1:8000/accounts/reset/...) y pégala en tu navegador.
  6. Establece tu nueva contraseña.

✅ Resultado

Si sigues estos pasos, habrás verificado el ciclo completo de autenticación y recuperación sin necesidad de un servidor de correo real.
